# 后台并行重构看板

## 模块拆分

### 1. 客户与权限
- 范围：客户列表、客户详情、联系人/门户成员、余额流水、客户标签、客户权限视图。
- 目标：对齐魔方式客户工作台，补齐 tabs、批量动作、高级筛选和角色权限。
- 当前状态：已完成基础列表页和详情页骨架，待补筛选、批量动作、客户分层和更多详情分区。

### 2. 订单与财务链
- 范围：订单、账单、支付、退款、财务任务、对账入口。
- 目标：稳定 `订单 -> 账单 -> 支付/退款 -> 服务` 主链，补足详情工作台和运营动作。
- 当前状态：已完成基础列表页、详情页和部分财务动作，待补高级筛选、更多审批动作和财务对象联动。

### 3. 服务、资源与魔方云
- 范围：服务列表、服务详情、资源对象、魔方云同步、实例动作、资源级动作。
- 目标：把服务域做成真正的实例工作台，并把资源级能力和同步差异补完整。
- 当前状态：已完成实例动作入口和服务详情骨架，待补资源级视图、差异同步和状态闭环。

### 4. 门户、工单、通知与审计
- 范围：客户门户、工单协同、通知队列、审计留痕。
- 目标：让后台和门户形成统一业务闭环，保证客户动作、通知和后台留痕一致。
- 当前状态：已完成门户主链和部分通知/工单能力，待补细化的自助能力和闭环验证。

- 汇总各模块的 IA 差距。
- 确认统一的数据模型边界。
- 生成下一阶段实施顺序。

## 模块结论汇总

### 客户与权限
- 当前成熟度：约 50%，有客户工作台骨架，但还没有对齐魔方式客户模块。
- 核心缺口：
  - 缺实名认证、资源池、客户分组、客户等级、联系人、跟进记录、营销推送、推介计划。
  - 客户详情缺联系人/门户权限、实名认证、跟进、交易/收款、退款、通知、账务时间线。
  - 列表缺分页、高级筛选、批量动作。
  - 权限仍偏粗，只做到 `customers.manage` 这一层。
- 最高优先级：
  - 拆出独立的客户一级模块。
  - 补客户域核心模型。
  - 把客户详情改成完整工作台。
  - 补分页、高级筛选、批量动作。
  - 把权限拆成细粒度 RBAC。

### 订单与财务链
- 当前成熟度：约 70%，主链已拆开，但还没形成真正财务闭环。
- 核心缺口：
  - 收款/退款列表缺筛选、分页、批量动作。
  - 订单和账单列表筛选不足。
  - 缺支付详情页、退款详情页。
  - 订单详情和账单详情缺明确动作区。
  - 还缺独立交易流水/账户余额视图。
- 最高优先级：
  - 把账单详情补成财务工作台。
  - 新增支付详情和退款详情。
  - 补齐订单/账单/收款退款列表页高级筛选。
  - 把订单详情补成生命周期工作区。
  - 增加交易流水/账户余额视图。

### 服务、资源与魔方云
- 当前成熟度：
  - 服务域：中等偏上，商用骨架已成。
  - 魔方云对接：中等偏低，仍是“实例级半实装 + 资源级只读拉取 + 部分 mock”。
- 核心缺口：
  - 真实 API 契约还没完全按文档锁定。
  - 资源级写能力缺失。
  - IP/VPC/磁盘扩容/配置变更/续费规则未接入。
  - 同步仍偏全量覆盖，缺差异同步。
  - 服务状态模型粒度太粗。
  - 缺任务态、锁定态、救援态持久化。
- 最高优先级：
  - 把实例生命周期真实对接做实。
  - 补齐资源级 provider 能力。
  - 重构服务状态模型。
  - 把同步改成真正差异同步。
  - 把配置变更和续费规则接入主链。

### 门户、工单、通知与审计
- 当前成熟度：已有基础一体化，但后台协同、事件回推和门户自助还没闭环。
- 核心缺口：
  - 工单缺部门、SLA、预设回复、附件、合并工单、派单规则。
  - 工单状态变化没有完整通知回推。
  - 通知中心还不是完整消息基础设施。
  - 审计缺对象级时间线和链路聚合。
  - 门户缺充值、账单详情、合并支付、开票/合同、退款进度。
  - 门户缺 API 管理、实名与安全、联系人/子账号、公告帮助。
- 最高优先级：
  - 打通工单闭环。
  - 把通知中心做成真实消息基础设施。
  - 建统一审计事件模型。
  - 补齐门户财务自助最小闭环。
  - 补齐门户核心自助能力。

## 统一实施顺序

### Wave 1
- 客户：独立一级模块、细粒度权限、列表高级筛选、批量动作。
- 财务：账单详情工作台、支付详情、退款详情。
- 服务：状态模型重构、实例生命周期真实对接。
- 门户/协同：工单闭环、统一通知和审计事件。

### Wave 2
- 客户：实名认证、联系人、跟进、资源池。
- 财务：交易流水、账户余额工作台、对账强化。
- 服务：资源级真实 API、差异同步、配置变更与续费规则。
- 门户：充值、合并支付、开票/合同、退款进度。

### Wave 3
- 客户：营销推送、推介计划、客户分层运营。
- 财务：更多审批、核销、红冲和财务报表。
- 服务：安全组/VPC/IP/磁盘全动作闭环。
- 门户：API 管理、安全中心、联系人/子账号、帮助下载。
